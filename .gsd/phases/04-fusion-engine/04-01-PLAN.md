  <action>
Create `src/gold_signal_bot/analysis/fusion.py` with:

1. Define `IndicatorWeight` dataclass for weight configuration:
   ```python
   @dataclass
   class IndicatorWeight:
       rsi: float = 0.25      # 25% - momentum
       macd: float = 0.30     # 30% - trend + momentum
       ema: float = 0.25      # 25% - trend
       bollinger: float = 0.20  # 20% - volatility/mean reversion
   ```

2. Define `FusionResult` dataclass:
   ```python
   @dataclass
   class FusionResult:
       bullish_score: float   # 0.0 to 1.0
       bearish_score: float   # 0.0 to 1.0
       direction: SignalDirection  # BULLISH if bullish > bearish, etc.
       aligned_indicators: list[str]  # Names of aligned indicators
       conflicting_indicators: list[str]  # Names that conflict
       sr_bonus: float  # S/R proximity bonus applied
   ```

3. Create `FusionEngine` class:
   - Constructor takes optional `IndicatorWeight` (default if not provided)
   - Method `fuse(snapshot: TechnicalSnapshot, current_price: float, support: PriceLevel | None, resistance: PriceLevel | None) -> FusionResult`
   
4. Fusion logic in `fuse()`:
   - For each indicator in snapshot (rsi, macd, ema, bollinger):
     - If signal == BULLISH: add weight to bullish_score, track as aligned
     - If signal == BEARISH: add weight to bearish_score, track as conflicting (if opposite)
     - If signal == NEUTRAL: no contribution
   - Apply S/R bonus:
     - If direction will be BULLISH and support is within 1% of price: +0.10 to bullish_score
     - If direction will be BEARISH and resistance is within 1% of price: +0.10 to bearish_score
   - Determine final direction based on which score is higher
   - Cap scores at 1.0

5. Update `__init__.py` to export FusionEngine, IndicatorWeight, FusionResult

  </verify>
  <done>

    - S/R proximity bonus applied correctly

  </done>
